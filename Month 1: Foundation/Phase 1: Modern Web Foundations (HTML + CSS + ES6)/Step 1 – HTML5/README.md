# Step 1 – HTML5 (Days 1 – 2)

## Learning Objectives

### 1. Semantic HTML Elements
Learn to use semantic elements for better document structure and accessibility:
- `<header>` - Page or section header
- `<main>` - Main content area
- `<section>` - Thematic grouping of content
- `<footer>` - Page or section footer
- `<nav>` - Navigation links
- `<article>` - Self-contained content
- `<aside>` - Sidebar content

### 2. Forms and Input Elements
Master form creation with proper structure and accessibility:
- Form elements (`<form>`, `<input>`, `<textarea>`, `<select>`)
- Labels and their association with inputs
- Input types (text, email, password, number, date, etc.)
- Accessibility attributes (`aria-*`, `role`, `aria-label`, `aria-describedby`)
- Form validation attributes (`required`, `pattern`, `min`, `max`)

### 3. Media Elements
Work with modern media elements:
- `<video>` - Embed video content
- `<audio>` - Embed audio content
- `<picture>` - Responsive images with multiple sources
- `<source>` - Define multiple media resources
- Media attributes (controls, autoplay, loop, poster)

### 4. SEO and Metadata
Understand essential meta tags and SEO basics:
- `<title>` - Page title
- `<meta>` tags (description, keywords, viewport)
- Open Graph tags for social media
- Favicons and app icons
- Structured data basics

## Practice Project: Portfolio Landing Page

Build a **simple portfolio landing page** using **only HTML5**.

### Requirements:
- ✅ Use semantic HTML5 elements throughout
- ✅ Include a header with navigation
- ✅ Create sections for:
  - About/Introduction
  - Skills or Services
  - Projects (at least 3)
  - Contact form
- ✅ Implement proper form with labels and accessibility
- ✅ Add appropriate meta tags for SEO
- ✅ Include a favicon
- ✅ Ensure all content is semantic and accessible

### Key Points to Demonstrate:
- Proper document structure
- Semantic markup
- Form accessibility
- SEO best practices
- Valid HTML5 code

### Validation:
- Use [W3C Markup Validation Service](https://validator.w3.org/) to ensure valid HTML
- Check accessibility with browser DevTools
